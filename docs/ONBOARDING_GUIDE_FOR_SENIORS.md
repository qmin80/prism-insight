## 온보딩 전략 개요

### 핵심 원칙

1. **점진적 복잡도 증가**: 간단한 것부터 복잡한 것까지 순차적으로 학습
2. **실습 중심**: 이론보다 실습을 통한 이해
3. **문서 기반**: 모든 설명이 문서화되어 있어 자율 학습 가능
4. **빠른 피드백**: 작은 단위로 확인하고 피드백 제공

### 예상 소요 시간

- **1주차**: 환경 설정 및 기본 개념 이해 (20-30시간)
- **2주차**: 핵심 모듈 이해 및 간단한 수정 (30-40시간)
- **3주차**: 기능 추가 및 버그 수정 (40-50시간)
- **4주차**: 독립적인 기능 개발 가능 (50-60시간)

### Week 1: 환경 설정 및 기본 이해

#### Day 1-2: 프로젝트 개요 및 환경 설정

**목표**: 프로젝트 전반을 이해하고 개발 환경을 구축

**활동**:
1. **문서 읽기** (2-3시간)
   - `docs/ONBOARDING_ko.md` 1-3장 읽기
   - 프로젝트 구조 파악
   - 기술 스택 이해

2. **환경 설정** (3-4시간)
   ```bash
   # 저장소 클론
   git clone <repository-url>
   cd prism-insight
   
   # 가상환경 설정
   python3 -m venv venv
   source venv/bin/activate  # Linux/Mac
   # 또는
   venv\Scripts\activate  # Windows
   
   # 의존성 설치
   pip install -r requirements.txt
   
   # Playwright 설치
   cd utils
   ./setup_playwright.sh
   ```

3. **설정 파일 준비** (1-2시간)
   - `.env` 파일 생성 (예시 파일 참고)
   - `mcp_agent.config.yaml` 설정
   - `mcp_agent.secrets.yaml` 설정 (API 키 등)
   - `trading/config/kis_devlp.yaml` 설정 (모의투자용)

4. **간단한 테스트 실행** (1-2시간)
   ```bash
   # 시장 영업일 체크
   python check_market_day.py
   
   # 종목 정보 업데이트
   python update_stock_data.py
   ```

**확인 사항**:
- [ ] Python 환경이 정상적으로 작동하는가?
- [ ] 모든 의존성이 설치되었는가?
- [ ] 설정 파일이 올바르게 구성되었는가?
- [ ] 기본 스크립트가 실행되는가?

**시니어 체크포인트**:
- 환경 설정 중 문제 발생 시 즉시 해결 지원
- 설정 파일 보안 주의사항 안내

## FAQ 및 트러블슈팅

### 자주 묻는 질문

#### Q1: 어디서부터 시작해야 할까요?

**A**: 다음 순서를 권장합니다:
1. `docs/ONBOARDING_ko.md` 1-3장 읽기
2. 환경 설정 완료
3. `cores/main.py` 실행하여 단일 종목 분석 테스트
4. 생성된 리포트 확인
5. 코드를 따라가며 데이터 흐름 이해

#### Q2: 코드가 너무 복잡해 보입니다. 어떻게 이해해야 할까요?

**A**: 다음 전략을 사용하세요:
1. **상단에서 하단으로**: 전체 흐름을 먼저 이해
2. **작은 단위로**: 한 번에 하나의 함수만 집중
3. **문서 활용**: 각 파일의 상단 설명 읽기
4. **디버깅**: breakpoint를 설정하고 실행 흐름 추적
5. **질문**: 이해가 안 되면 즉시 질문

#### Q3: 에러가 발생했을 때 어떻게 해결해야 할까요?

**A**: 다음 순서로 접근하세요:
1. **로그 확인**: 에러 메시지와 스택 트레이스 확인
2. **문서 확인**: 관련 문서에서 해결 방법 찾기
3. **코드 추적**: 에러가 발생한 위치의 코드 확인
4. **검색**: 비슷한 에러가 있는지 검색
5. **질문**: 해결되지 않으면 시니어에게 질문

#### Q4: AI 에이전트가 어떻게 작동하는지 이해하기 어렵습니다.

**A**: 다음 단계로 학습하세요:
1. **단일 에이전트부터**: `cores/agents/stock_price_agents.py` 하나만 집중
2. **프롬프트 분석**: instruction 문자열 분석
3. **실행 테스트**: 에이전트를 단독으로 실행하여 출력 확인
4. **도구 사용 확인**: 어떤 MCP 서버를 사용하는지 확인
5. **점진적 확장**: 여러 에이전트의 협업 방식 이해

#### Q5: 실제 매매가 실행되는지 걱정됩니다.

**A**: 다음 안전장치를 활용하세요:
1. **모의투자 모드**: `trading/config/kis_devlp.yaml`에서 `default_mode: "demo"` 설정
2. **자동 매매 비활성화**: `auto_trading: false` 설정
3. **소액 테스트**: 모의투자에서 소액으로 테스트
4. **로그 확인**: 모든 매매 결정이 로그에 기록됨

### 트러블슈팅 가이드

#### 문제: Python 패키지 설치 실패

**해결 방법**:
```bash
# Python 버전 확인 (3.10+ 필요)
python3 --version

# pip 업그레이드
pip install --upgrade pip

# 가상환경 재생성
rm -rf venv
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

#### 문제: Playwright 브라우저 설치 실패

**해결 방법**:
```bash
# Playwright 재설치
pip install --upgrade playwright
python3 -m playwright install chromium

# 또는 스크립트 사용
cd utils
./setup_playwright.sh
```

#### 문제: API 키 오류

**해결 방법**:
1. `.env` 파일 확인
2. `mcp_agent.secrets.yaml` 파일 확인
3. API 키가 올바른 형식인지 확인
4. API 키에 따옴표가 없는지 확인

#### 문제: 데이터베이스 연결 오류

**해결 방법**:
```bash
# 데이터베이스 파일 확인
ls -la stock_tracking_db.sqlite

# 권한 확인
chmod 644 stock_tracking_db.sqlite

# SQLite로 직접 연결 테스트
sqlite3 stock_tracking_db.sqlite
```

#### 문제: 텔레그램 메시지 전송 실패

**해결 방법**:
1. `TELEGRAM_BOT_TOKEN` 확인
2. `TELEGRAM_CHANNEL_ID` 확인
3. 봇이 채널에 추가되어 있는지 확인
4. 봇 권한 확인 (메시지 전송 권한)
